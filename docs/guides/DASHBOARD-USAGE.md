# Dashboard Usage Guide

**Complete guide to using the Lev8 Analytics Dashboard**

---

## Overview

The Analytics Dashboard (Understand Module) provides comprehensive resident evaluation analytics with three main views:

- **Individual Resident:** Detailed analytics for a single resident
- **Class View:** Aggregated analytics for a cohort (Coming Soon)
- **Program View:** Program-wide trends and statistics (Coming Soon)

---

## Accessing the Dashboard

### For Program Directors and Faculty

1. Log in to Lev8
2. Navigate to "Understand" in the sidebar
3. Click "Overview"
4. Select "Individual Resident"
5. Choose a resident from the dropdown
6. Click "View Analytics"

### For Residents

1. Log in to Lev8
2. Navigate to "Understand" in the sidebar
3. Click "Overview"
4. Your analytics will display automatically (residents can only see their own data)

## Dashboard Tabs

### 1. SWOT Analysis Tab

**What it shows:**
- AI-generated Strengths, Weaknesses, Opportunities, and Threats
- Based on faculty evaluation comments from MedHub
- Brutally honest feedback for program director review
- Supporting quotes with citations

**Features:**

**Period Selector:**
- Filter by academic period (e.g., "PGY-2 Fall", "PGY-3 Spring")
- Dropdown shows all available periods for the resident
- Updates all tabs when changed

**SWOT Cards:**
- Four color-coded cards: Strengths (Green), Weaknesses (Red), Opportunities (Blue), Threats (Orange)
- Each card shows 3-5 key themes
- Frequency badges (e.g., "3Ã—") indicate how often theme appeared
- Severity levels for weaknesses (CRITICAL, MODERATE, MINOR)

**Citations:**
- Click "See Citations" to expand supporting quotes
- Each quote includes:
  - Direct text from faculty evaluation
  - Citation (e.g., "Faculty Comment 12 (2024-02-15)")
- Use these specific examples in feedback conversations

**How to Use:**

1. **Review Strengths:**
   - Highlight in feedback sessions
   - Encourage resident to continue these behaviors
   - Use as confidence builders

2. **Address Weaknesses:**
   - Note severity levels (CRITICAL requires immediate action)
   - Create improvement plans
   - Set specific, measurable goals
   - Schedule follow-up

3. **Explore Opportunities:**
   - Suggest specific rotations
   - Recommend learning resources
   - Connect with mentors in these areas

4. **Monitor Threats:**
   - Watch for warning signs
   - Provide proactive support
   - May require intervention or remediation

### 2. EQ + PQ + IQ Tab

**What it shows:**
- Emotional, Professional, and Intellectual Quotient scores
- 15-point radar chart visualization
- Faculty vs Self-assessment comparison
- Gap analysis
- ITE score history

**Features:**

**15-Point Radar Chart:**
- Visualizes all 15 attributes on one comprehensive chart
- Color-coded sections:
  - **EQ (Pink):** Empathy, Adaptability, Stress Management, Curiosity, Communication
  - **PQ (Teal):** Work Ethic, Integrity, Teachability, Documentation, Leadership
  - **IQ (Green):** Knowledge Base, Analytical Thinking, Learning, Flexibility, Performance
- Faculty data (blue line) overlaid with Self data (pink line)
- Interactive tooltips on hover
- Scale: 1.0-5.0

**How to Read the Radar Chart:**

1. **Look for Shape:**
   - Balanced shape = well-rounded resident
   - Spikes = exceptional in specific areas
   - Dips = areas needing improvement

2. **Compare Faculty vs Self:**
   - Overlapping lines = good self-awareness
   - Self higher than faculty = potential overconfidence
   - Faculty higher than self = potential underconfidence

3. **Identify Patterns:**
   - All EQ attributes low = emotional intelligence focus needed
   - All PQ attributes high = strong professionalism
   - All IQ attributes variable = inconsistent clinical performance

**Gap Analysis:**
- Shows difference between self and faculty perception
- Three metrics: EQ gap, PQ gap, IQ gap
- Color-coded:
  - **Green:** Small gap (< 0.5) - Good self-awareness
  - **Yellow:** Moderate gap (0.5-1.0) - Some blind spots
  - **Red:** Large gap (> 1.0) - Significant perception mismatch

**Interpreting Gaps:**
- **Positive Gap (Self > Faculty):** Resident may be overconfident
- **Negative Gap (Faculty > Self):** Resident may be underconfident or overly self-critical
- **Large Gaps:** Indicate blind spots; use in coaching conversations

**Detailed Scores Table:**
- Shows exact values for all 15 attributes
- Faculty average (from multiple evaluators)
- Self-assessment value
- Number of faculty raters

**ITE Scores:**
- Historical exam performance
- Percentile rankings
- Trend over time
- Compare to national averages

**How to Use:**

1. **In Feedback Sessions:**
   - Show resident their radar chart
   - Discuss specific attributes, not just averages
   - Use gap analysis to explore self-perception
   - Celebrate improvements from previous periods

2. **For Goal Setting:**
   - Identify 2-3 lowest attributes
   - Create specific action plans
   - Set target scores for next period
   - Track progress over time

3. **For Competency Committees:**
   - Use as objective data for discussions
   - Compare to program benchmarks
   - Identify residents needing support
   - Track longitudinal trends

## Common Use Cases

### Use Case 1: Semi-Annual Review

**Scenario:** Program director preparing for resident's semi-annual review.

**Steps:**
1. Navigate to resident's analytics
2. Select current period (e.g., "PGY-2 Fall")
3. Review SWOT tab:
   - Note 2-3 key strengths to celebrate
   - Identify 1-2 critical weaknesses to address
   - Discuss opportunities for growth
4. Review Scores tab:
   - Show radar chart to resident
   - Discuss gap analysis
   - Compare to previous period
5. Create action plan with specific goals
6. Document in evaluation system

---

### Use Case 2: Identifying At-Risk Residents

**Scenario:** Program director wants to identify residents who may need intervention.

**Steps:**
1. Review multiple residents' analytics
2. Look for red flags:
   - Multiple CRITICAL weaknesses
   - Declining ITE scores
   - Large negative gaps (faculty much lower than self)
   - Consistently low EQ/PQ/IQ scores (< 3.0)
3. Schedule one-on-one meetings
4. Create remediation plans if needed
5. Monitor progress closely

---

### Use Case 3: Resident Self-Reflection

**Scenario:** Resident reviewing their own performance.

**Steps:**
1. Log in and navigate to own analytics
2. Review SWOT analysis:
   - Acknowledge strengths
   - Accept weaknesses honestly
   - Identify opportunities to pursue
3. Review radar chart:
   - Compare self-assessment to faculty feedback
   - Identify blind spots (large gaps)
   - Recognize areas of growth
4. Set personal goals for next period
5. Seek mentorship in weak areas

---

### Use Case 4: Faculty Calibration

**Scenario:** Program wants to ensure faculty are rating consistently.

**Steps:**
1. Review multiple residents evaluated by same faculty
2. Look for patterns:
   - Faculty who rate everyone high (grade inflation)
   - Faculty who rate everyone low (overly critical)
   - Inconsistent ratings across similar residents
3. Provide faculty development on evaluation standards
4. Use rubrics to improve consistency

## Tips for Effective Use

### For Program Directors

1. **Review Regularly:**
   - Check analytics monthly, not just at evaluation time
   - Track trends over time
   - Identify issues early

2. **Use Specific Examples:**
   - Reference supporting quotes in feedback
   - Avoid generalizations
   - Be concrete and actionable

3. **Balance Positive and Negative:**
   - Start with strengths
   - Frame weaknesses as growth opportunities
   - End with encouragement

4. **Create Action Plans:**
   - Set specific, measurable goals
   - Assign resources (mentors, readings, rotations)
   - Schedule follow-up
   - Document progress

5. **Protect Confidentiality:**
   - SWOT analysis is for program director review
   - Share appropriately with residents
   - Maintain faculty anonymity

---

### For Faculty

1. **Submit Evaluations Regularly:**
   - Don't wait until end of rotation
   - Provide timely feedback
   - Be specific in comments

2. **Use Full Rating Scale:**
   - Don't cluster around 3.0
   - Use 1.0-2.0 for significant concerns
   - Use 4.5-5.0 for exceptional performance

3. **Provide Context:**
   - Include specific examples
   - Note circumstances (busy shift, complex case)
   - Mention improvement or decline

4. **Be Honest:**
   - Don't inflate ratings
   - Address concerns directly
   - Focus on behaviors, not personality

---

### For Residents

1. **Be Honest in Self-Assessments:**
   - Accurate self-perception helps identify blind spots
   - Acknowledge weaknesses
   - Don't inflate scores

2. **Review Regularly:**
   - Check analytics each period
   - Track your progress
   - Celebrate improvements

3. **Seek Clarification:**
   - If feedback surprises you, ask for examples
   - Understand the "why" behind ratings
   - Request specific improvement strategies

4. **Act on Feedback:**
   - Create personal development plan
   - Seek mentorship
   - Follow through on goals

## Troubleshooting

### No Data Showing

**Problem:** Dashboard is empty

**Solutions:**
1. Verify data exists in database:
   ```sql
   SELECT COUNT(*) FROM swot_summaries WHERE resident_id = 'your-id';
   SELECT COUNT(*) FROM period_scores WHERE resident_id = 'your-id';
   ```
2. Check that period selector has options
3. Verify you have permission to view this resident
4. Try a different period

---

### Radar Chart Not Displaying

**Problem:** Chart shows "Loading..." or error

**Solutions:**
1. Verify `recharts` is installed: `npm list recharts`
2. Check browser console for errors
3. Verify period has data in `period_scores`
4. Try refreshing the page

---

### Citations Not Expanding

**Problem:** "See Citations" button doesn't work

**Solutions:**
1. Check browser console for JavaScript errors
2. Verify `supporting_quotes` exist in database
3. Try a different browser
4. Clear browser cache
